# P2P Chat - Лабораторная работа 3

## Описание
Программа для обмена текстовыми сообщениями в локальной сети в одноранговом (P2P) режиме.

## Затрагиваемые темы
- Программирование клиент-серверного приложения
- Протоколы TCP и UDP, разработка собственного протокола
- Многопоточное программирование

## Требования
- Python 3.6+
- Сокеты (встроенная библиотека)
- Потоки (встроенная библиотека)

## Запуск программы

### Синтаксис
```bash
python3 p2p_chat.py <Имя_узла> <IP_адрес>
```

### Примеры запуска нескольких узлов на одном компьютере
```bash
# Терминал 1
python3 p2p_chat.py User1 127.0.0.1

# Терминал 2
python3 p2p_chat.py User2 127.0.0.2

# Терминал 3
python3 p2p_chat.py User3 127.0.0.3
```

## Архитектура приложения

### Используемые протоколы

#### UDP (порт 5000)
- **Назначение**: Обнаружение узлов в сети (discovery)
- **Тип трафика**: Multicast (группа 224.0.0.1)
- **Формат пакета**: `<Имя>|<IP>` (строка UTF-8)

#### TCP (порт 5000)
- **Назначение**: Обмен сообщениями между узлами
- **Тип соединения**: Полносвязная топология (каждый с каждым)
- **Формат сообщения**: Собственный бинарный протокол

### Формат сообщения (собственный протокол)

| Смещение | Размер | Описание |
|----------|--------|----------|
| 0        | 1 байт | Тип сообщения |
| 1        | 1 байт | Длина данных (n) |
| 2        | n байт | Данные сообщения |

#### Типы сообщений

| Код | Тип | Описание | Направление |
|-----|-----|----------|-------------|
| 1   | MSG | Текстовое сообщение чата | Все → Все |
| 2   | NAME | Передача имени узла | Новый → Существующие |
| 3   | JOIN | Узел подключен | Система |
| 4   | LEAVE | Узел отключен | Система |
| 5   | HIST_REQ | Запрос истории | Новый → Существующий |
| 6   | HIST_DATA | Данные истории | Существующий → Новый |

## Принцип работы

### 1. Обнаружение узлов (UDP Discovery)
1. При запуске узел периодически отправляет multicast-пакет со своим именем и IP
2. Другие узлы получают пакет и устанавливают TCP-соединение с новым узлом
3. При установке соединения происходит обмен именами

### 2. Подключение нового узла
1. Новый узел получает UDP-пакеты от существующих узлов
2. Устанавливает TCP-соединения со всеми активными узлами
3. Отправляет запрос истории (тип 5) одному из узлов
4. Получает полную историю сообщений (тип 6)

### 3. Обмен сообщениями
1. Пользователь вводит сообщение
2. Сообщение отправляется всем подключенным узлам через TCP
3. Каждый узел отображает сообщение в общем чате

### 4. Отключение узла
1. При корректном закрытии соединения узлы фиксируют отключение
2. В историю добавляется запись об отключении
3. Соединение удаляется из списка пиров

## Тестирование

### Шаг 1: Подготовка
Запустите несколько экземпляров программы с разными IP адресами loopback интерфейса:

```bash
# Окно 1
python3 p2p_chat.py Alice 127.0.0.1

# Окно 2  
python3 p2p_chat.py Bob 127.0.0.2

# Окно 3
python3 p2p_chat.py Charlie 127.0.0.3
```

### Шаг 2: Проверка обнаружения узлов
Убедитесь, что каждый узел обнаружил остальные:
- В логах должно появиться "Обнаружен узел: <имя> (<IP>)"
- В логах должно появиться "Подключение к <имя> (<IP>)"

### Шаг 3: Проверка передачи истории
Последний запущенный узел должен получить историю от предыдущих узлов.

### Шаг 4: Отправка сообщений
Введите сообщение в любом узле - оно должно отобразиться во всех узлах.

### Шаг 5: Проверка отключения
Закройте один из узлов (Ctrl+C) - остальные должны зафиксировать отключение.

### Шаг 6: Анализ трафика в Wireshark

#### Фильтры для анализа:
- `udp.port == 5000` - discovery пакеты
- `tcp.port == 5000` - обмен сообщениями
- `ip.dst == 224.0.0.1` - multicast трафик

#### Что должно быть видно:
1. **UDP broadcast/multicast**: Периодические пакеты обнаружения
2. **TCP handshake**: Трехкратное рукопожатие при подключении каждого узла
   - SYN → SYN-ACK → ACK
3. **Передача сообщений**: TCP пакеты с данными
4. **Отключение**: Флаги FIN или RST при закрытии соединения

## Вопросы для защиты

### 1. Понятие порта
Порт - это 16-битное число (0-65535), используемое для идентификации конкретного процесса или службы на хосте в сочетании с IP-адресом. Позволяет множеству приложений работать на одном хосте одновременно.

### 2. Протокол UDP
**Характеристики**:
- Негарантированная доставка
- Без установления соединения
- Минимальные накладные расходы

**Структура UDP-пакета** (8 байт заголовок):
| Поле | Размер | Описание |
|------|--------|----------|
| Порт отправителя | 2 байта | Порт источника |
| Порт получателя | 2 байта | Порт назначения |
| Длина | 2 байта | Длина заголовка + данных |
| Контрольная сумма | 2 байта | CRC для проверки целостности |

### 3. Протокол TCP
**Характеристики**:
- Гарантированная доставка
- Установка соединения перед передачей данных
- Контроль потока и перегрузки

**Структура TCP-пакета** (минимум 20 байт):
| Поле | Размер | Описание |
|------|--------|----------|
| Порт отправителя | 2 байта | Порт источника |
| Порт получателя | 2 байта | Порт назначения |
| Номер последовательности | 4 байта | Порядковый номер байта |
| Номер подтверждения | 4 байта | Ожидаемый следующий байт |
| Смещение заголовка | 4 бита | Длина заголовка |
| Флаги | 8 бит | SYN, ACK, FIN, RST и др. |
| Размер окна | 2 байта | Размер буфера приема |
| Контрольная сумма | 2 байта | CRC |
| Срочный указатель | 2 байта | Для urgent данных |

**Трехкратное рукопожатие**:
1. Клиент → Сервер: SYN (seq=x)
2. Сервер → Клиент: SYN-ACK (seq=y, ack=x+1)
3. Клиент → Сервер: ACK (ack=y+1)

**Механизм скользящего окна**:
- Позволяет отправлять несколько сегментов без подтверждения каждого
- Размер окна динамически изменяется на основе загруженности сети
- Повышает пропускную способность соединения

### 4. Гнезда (сокеты) UDP и TCP

**UDP сокет**:
```python
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
sock.bind(('0.0.0.0', port))
# Отправка
sock.sendto(data, (ip, port))
# Получение
data, addr = sock.recvfrom(1024)
```

**TCP сокет (сервер)**:
```python
sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
sock.bind((ip, port))
sock.listen(5)
conn, addr = sock.accept()
```

**TCP сокет (клиент)**:
```python
sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
sock.connect((ip, port))
```

### 5. Маска подсети

**Определение**: Маска подсети определяет, какая часть IP-адреса относится к сети, а какая к узлу.

**Примеры**:
| Маска | CIDR | Бинарная маска | Сеть | Узлов |
|-------|------|----------------|------|-------|
| 255.0.0.0 | /8 | 11111111.00000000.00000000.00000000 | 1 байт | 2^24 - 2 = 16,777,214 |
| 255.255.0.0 | /16 | 11111111.11111111.00000000.00000000 | 2 байта | 2^16 - 2 = 65,534 |
| 255.255.255.0 | /24 | 11111111.11111111.11111111.00000000 | 3 байта | 2^8 - 2 = 254 |
| 255.255.255.128 | /25 | 11111111.11111111.11111111.10000000 | 25 бит | 2^7 - 2 = 126 |
| 255.255.255.192 | /26 | 11111111.11111111.11111111.11000000 | 26 бит | 2^6 - 2 = 62 |

**Расчет количества узлов**: `2^(32 - количество_единиц_в_маске) - 2`
- Вычитается 2: адрес сети и широковещательный адрес

**Пример для 127.0.0.0/8**:
- Маска: 255.0.0.0
- Номер сети: 127
- Номер узла: последние 3 октета
- Количество узлов: 16,777,214
